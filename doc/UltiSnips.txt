Description                                        *UltiSnips-description*</div><div class='line' id='LC53'><br/></div><div class='line' id='LC54'>UltiSnips provides snippet management for the Vim editor. A snippet is a short</div><div class='line' id='LC55'>piece of text that is either re-used often or contains a lot of redundant</div><div class='line' id='LC56'>text. UltiSnips allows you to insert a snippet with only a few key strokes.</div><div class='line' id='LC57'>Snippets are common in structured text like source code but can also be used</div><div class='line' id='LC58'>for general editing like, for example, inserting a signature in an email or</div><div class='line' id='LC59'>inserting the current date in a text file.</div><div class='line' id='LC60'><br/></div><div class='line' id='LC61'>UltiSnips was developed using the TDD (Test-driven development) philosophy.</div><div class='line' id='LC62'>This ensures that features do not disappear and bugs do not reappear after</div><div class='line' id='LC63'>they have been fixed.</div><div class='line' id='LC64'><br/></div><div class='line' id='LC65'>On my blog, http://www.sirver.net, I posted several short screencasts, which</div><div class='line' id='LC66'>make a great introduction to UltiSnips, illustrating its features and usage.</div><div class='line' id='LC67'><br/></div><div class='line' id='LC68'>http://www.sirver.net/blog/2011/12/30/first-episode-of-ultisnips-screencast/</div><div class='line' id='LC69'>http://www.sirver.net/blog/2012/01/08/second-episode-of-ultisnips-screencast/</div><div class='line' id='LC70'>http://www.sirver.net/blog/2012/02/05/third-episode-of-ultisnips-screencast/</div><div class='line' id='LC71'>http://www.sirver.net/blog/2012/03/31/fourth-episode-of-ultisnips-screencast/</div><div class='line' id='LC72'><br/></div><div class='line' id='LC73'><br/></div><div class='line' id='LC74'>1.1 Requirements                                     *UltiSnips-requirements*</div><div class='line' id='LC75'>----------------</div><div class='line' id='LC76'><br/></div><div class='line' id='LC77'>This plugin works with Vim version 7.0 or later. It only works if the</div><div class='line' id='LC78'>&#39;compatible&#39; setting is not set.</div><div class='line' id='LC79'><br/></div><div class='line' id='LC80'>This plugin requires python &gt;= 2.6. In other words, Vim</div><div class='line' id='LC85'>must be compiled with either the |+python| feature or the |+python3| feature.</div><div class='line' id='LC86'>The following commands show how to test if you have python compiled in Vim.</div><div class='line' id='LC87'>They print &#39;1&#39; if the python version is compiled in, &#39;0&#39; if not.</div><div class='line' id='LC88'><br/></div><div class='line' id='LC89'>Test if Vim is compiled with python version 2.x: &gt;</div><div class='line' id='LC90'>&nbsp;&nbsp;&nbsp;&nbsp;:echo has(&quot;python&quot;)</div><div class='line' id='LC91'>The python version Vim is linked against can be found with: &gt;</div><div class='line' id='LC92'>&nbsp;&nbsp;&nbsp;&nbsp;:py import sys; Opens in the current window.</div><div class='line' id='LC189'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;|horizontal|     Splits the window horizontally.</div><div class='line' id='LC190'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;|vertical|       Splits the window vertically.</div><div class='line' id='LC191'><br/></div><div class='line' id='LC192'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*g:UltiSnipsSnippetsDir*</div><div class='line' id='LC193'>g:UltiSnipsSnippetsDir</div><div class='line' id='LC194'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Defines the directory private snippet definition</div><div class='line' id='LC195'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;files are stored in. Variables define the keys used to expand a snippet, jump forward</div><div class='line' id='LC229'>and jump backwards within a snippet, and list all available snippets in the</div><div class='line' id='LC230'>current expand context. The variables with their default values are: &gt;</div><div class='line' id='LC231'>&nbsp;&nbsp;&nbsp;g:UltiSnipsExpandTrigger               &lt;tab&gt;</div><div class='line' id='LC232'>&nbsp;&nbsp;&nbsp;g:UltiSnipsListSnippets                &lt;c-tab&gt;</div><div class='line' id='LC233'>&nbsp;&nbsp;&nbsp;g:UltiSnipsJumpForwardTrigger          &lt;c-j&gt;</div><div class='line' id='LC234'>&nbsp;&nbsp;&nbsp;g:UltiSnipsJumpBackwardTrigger         &lt;c-k&gt;</div><div class='line' id='LC235'><br/></div><div class='line' id='LC236'>The g:UltiSnipsExpandTrigger and g:UltiSnipsJumpForwardTrigger can be set to</div><div class='line' id='LC237'>the same value. To simulate TextMate behavior, add the following lines to your</div><div class='line' id='LC238'>vimrc file. &gt;</div><div class='line' id='LC239'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsExpandTrigger=&quot;&lt;tab&gt;&quot;</div><div class='line' id='LC240'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsJumpForwardTrigger=&quot;&lt;tab&gt;&quot;</div><div class='line' id='LC241'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsJumpBackwardTrigger=&quot;&lt;s-tab&gt;&quot;</div><div class='line' id='LC242'><br/></div><div class='line' id='LC243'>3.2.1 Using your own trigger functions           *UltiSnips-trigger-functions*</div><div class='line' id='LC244'>--------------------------------------</div><div class='line' id='LC245'><br/></div><div class='line' id='LC246'>For advanced users there are four functions that you can map directly to a</div><div class='line' id='LC247'>key and that correspond to some of the triggers previously defined:</div><div class='line' id='LC248'>&nbsp;&nbsp;&nbsp;g:UltiSnipsExpandTrigger        &lt;--&gt; UltiSnips_ExpandSnippet</div><div class='line' id='LC249'>&nbsp;&nbsp;&nbsp;g:UltiSnipsJumpForwardTrigger   &lt;--&gt; UltiSnips_JumpForwards</div><div class='line' id='LC250'>&nbsp;&nbsp;&nbsp;g:UltiSnipsJumpBackwardTrigger  &lt;--&gt; UltiSnips_JumpBackwards</div><div class='line' id='LC251'><br/></div><div class='line' id='LC252'>If you have g:UltiSnipsExpandTrigger and g:UltiSnipsJumpForwardTrigger set</div><div class='line' id='LC253'>to the same value then the function you are actually going to use is</div><div class='line' id='LC254'>UltiSnips_ExpandSnippetOrJump.</div><div class='line' id='LC255'><br/></div><div class='line' id='LC256'>Each time any of the functions UltiSnips_ExpandSnippet,</div><div class='line' id='LC257'>UltiSnips_ExpandSnippet, UltiSnips_JumpBackwards or UltiSnips_JumpBackwards is</div><div class='line' id='LC258'>called a global variable is set that contains the return value of the</div><div class='line' id='LC259'>corresponding function.</div><div class='line' id='LC260'><br/></div><div class='line' id='LC261'>The corresponding variables and functions are:</div><div class='line' id='LC262'>UltiSnips_ExpandSnippet       --&gt; g:ulti_expand_res (0: fail, 1: success)</div><div class='line' id='LC263'>UltiSnips_ExpandSnippetOrJump --&gt; g:ulti_expand_or_jump_res (0: fail,</div><div class='line' id='LC264'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: expand, 2: jump)</div><div class='line' id='LC265'>UltiSnips_JumpForwards        --&gt; g:ulti_jump_forwards_res (0: fail, 1: success)</div><div class='line' id='LC266'>UltiSnips_JumpBackwards       --&gt; g:ulti_jump_backwards_res (0: fail, 1: success)</div><div class='line' id='LC267'><br/></div><div class='line' id='LC268'>To see how these return values may come in handy, suppose that you want to map</div><div class='line' id='LC269'>a key to expand or jump, but if none of these actions is successful you want</div><div class='line' id='LC270'>to call another function. UltiSnips already does this automatically for</div><div class='line' id='LC271'>supertab, but this allows you individual fine tuning of your Tab key usage.</div><div class='line' id='LC272'><br/></div><div class='line' id='LC273'>Usage is as follows: You define a function &gt;</div><div class='line' id='LC274'><br/></div><div class='line' id='LC275'>&nbsp;&nbsp;&nbsp;let g:ulti_expand_or_jump_res = 0 &quot;default value, just set once</div><div class='line' id='LC276'>&nbsp;&nbsp;&nbsp;function! Ulti_ExpandOrJump_and_getRes()</div><div class='line' id='LC277'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;call UltiSnips_ExpandSnippetOrJump()</div><div class='line' id='LC278'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;return g:ulti_expand_or_jump_res</div><div class='line' id='LC279'>&nbsp;&nbsp;&nbsp;endfunction</div><div class='line' id='LC280'><br/></div><div class='line' id='LC281'>then you define your mapping as &gt;</div><div class='line' id='LC282'><br/></div><div class='line' id='LC283'>&nbsp;&nbsp;&nbsp;inoremap &lt;NL&gt; &lt;C-R&gt;=(Ulti_ExpandOrJump_and_getRes() &gt; 0)?&quot;&quot;:IMAP_Jumpfunc(&#39;&#39;, 0)&lt;CR&gt;</div><div class='line' id='LC284'><br/></div><div class='line' id='LC285'>and if the you can&#39;t expand or jump from the current location then the</div><div class='line' id='LC286'>alternative function IMAP_Jumpfunc(&#39;&#39;, 0) is called.</div><div class='line' id='LC287'><br/></div><div class='line' id='LC288'>3.2.2 Path to Python module                   *UltiSnips-python-module-path*</div><div class='line' id='LC289'>---------------------------</div><div class='line' id='LC290'><br/></div><div class='line' id='LC291'>For even more advanced usage, you can directly write python functions using</div><div class='line' id='LC292'>UltiSnip&#39;s python modules.</div><div class='line' id='LC293'><br/></div><div class='line' id='LC294'>The path to UltiSnip&#39;s python implementation is stored in the</div><div class='line' id='LC295'>g:UltiSnipsPythonPath variable which is set automatically on load.</div><div class='line' id='LC296'><br/></div><div class='line' id='LC297'>Here is a small example funtion that expands a snippet: &gt;</div><div class='line' id='LC298'><br/></div><div class='line' id='LC299'>&nbsp;&nbsp;&nbsp;function! s:Ulti_ExpandSnip()</div><div class='line' id='LC300'>&nbsp;&nbsp;&nbsp;Python &lt;&lt; EOF</div><div class='line' id='LC301'>&nbsp;&nbsp;&nbsp;import sys, vim</div><div class='line' id='LC302'>&nbsp;&nbsp;&nbsp;new_path = vim.eval(&quot;expand(&#39;g:UltiSnipsPythonPath&#39;)&quot;)</div><div class='line' id='LC303'>&nbsp;&nbsp;&nbsp;sys.path.append(new_path)</div><div class='line' id='LC304'>&nbsp;&nbsp;&nbsp;from UltiSnips import UltiSnips_Manager</div><div class='line' id='LC305'>&nbsp;&nbsp;&nbsp;UltiSnips_Manager.expand()</div><div class='line' id='LC306'>&nbsp;&nbsp;&nbsp;EOF</div><div class='line' id='LC307'>&nbsp;&nbsp;&nbsp;return &quot;&quot;</div><div class='line' id='LC308'>&nbsp;&nbsp;&nbsp;endfunction</div><div class='line' id='LC309'><br/></div><div class='line' id='LC310'>3.3 Snippet Search Path                       *UltiSnips-snippet-search-path*</div><div class='line' id='LC311'>-----------------------</div><div class='line' id='LC312'><br/></div><div class='line' id='LC313'>UltiSnips snippet definition files are stored in one or more directories.</div><div class='line' id='LC314'>There are several variables used to indicate those directories and to define</div><div class='line' id='LC315'>how UltiSnips loads snippets.</div><div class='line' id='LC316'><br/></div><div class='line' id='LC317'>Snippet definition files are stored in snippet directories. A snippet</div><div class='line' id='LC318'>directory must be a subdirectory of a directory defined in the &#39;runtimepath&#39;</div><div class='line' id='LC319'>option. The variable g:UltiSnipsSnippetDirectories defines a list of names</div><div class='line' id='LC320'>used for snippet directories. The default is shown below. &gt;</div><div class='line' id='LC321'><br/></div><div class='line' id='LC322'>&nbsp;&nbsp;&nbsp;g:UltiSnipsSnippetDirectories          [&quot;UltiSnips&quot;]</div><div class='line' id='LC323'><br/></div><div class='line' id='LC324'>UltiSnips will search each &#39;runtimepath&#39; directory for the subdirectory names</div><div class='line' id='LC325'>defined in g:UltiSnipsSnippetDirectories in the order they are defined. For</div><div class='line' id='LC326'>example, if you keep your snippets in a .vim subdirectory called</div><div class='line' id='LC327'>&quot;mycoolsnippets&quot; and you want to make use of the default snippets that come</div><div class='line' id='LC328'>with UltiSnips, add the following to your vimrc file. &gt;</div><div class='line' id='LC329'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsSnippetDirectories=[&quot;UltiSnips&quot;, &quot;mycoolsnippets&quot;]</div><div class='line' id='LC330'>If you do not want to use the snippets that come with UltiSnips, define the</div><div class='line' id='LC331'>variable accordingly. &gt;</div><div class='line' id='LC332'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsSnippetDirectories=[&quot;mycoolsnippets&quot;]</div><div class='line' id='LC333'><br/></div><div class='line' id='LC334'>You can also redefine the search path on a buffer by buffer basis by setting</div><div class='line' id='LC335'>the variable b:UltiSnipsSnippetDirectories.  This variable takes precedence</div><div class='line' id='LC336'>over the global variable.</div><div class='line' id='LC337'><br/></div><div class='line' id='LC338'>UltiSnips searches in &#39;runtimepath&#39; for snippet directories but traverses</div><div class='line' id='LC339'>&#39;runtimepath&#39; in reverse order (last item first). If you would like to have</div><div class='line' id='LC340'>UltiSnips traverse &#39;runtimepath&#39; in the standard order, add this to your vimrc</div><div class='line' id='LC341'>file: &gt;</div><div class='line' id='LC342'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsDontReverseSearchPath=&quot;1&quot;</div><div class='line' id='LC343'><br/></div><div class='line' id='LC344'>By default, whenever a snippet expand is triggered, UltiSnips will check for</div><div class='line' id='LC345'>modifications to the snippet file associated with the filetype and reload it</div><div class='line' id='LC346'>if necessary. This behavior can be disabled as follows: &gt;</div><div class='line' id='LC347'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsDoHash=0</div><div class='line' id='LC348'><br/></div><div class='line' id='LC349'>|UltiSnips-adding-snippets| explains which files are parsed for a given filetype.</div><div class='line' id='LC350'><br/></div><div class='line' id='LC351'><br/></div><div class='line' id='LC352'>3.4 Warning About Select Mode Mappings          *UltiSnips-warning-smappings*</div><div class='line' id='LC353'>--------------------------------------</div><div class='line' id='LC354'><br/></div><div class='line' id='LC355'>Vim&#39;s help document for |mapmode-s| states: &gt;</div><div class='line' id='LC356'>&nbsp;&nbsp;&nbsp;NOTE: Mapping a printable character in Select mode may confuse the user.</div><div class='line' id='LC357'>&nbsp;&nbsp;&nbsp;It&#39;s better to explicitly use :xmap and :smap for printable characters.  Or</div><div class='line' id='LC358'>&nbsp;&nbsp;&nbsp;use :sunmap after defining the mapping.</div><div class='line' id='LC359'><br/></div><div class='line' id='LC360'>However, most Vim plugins, including some default Vim plugins, do not adhere</div><div class='line' id='LC361'>to this. UltiSnips uses Select mode to mark tabstops in snippets for</div><div class='line' id='LC362'>overwriting. Existing Visual+Select mode mappings will interfere. Therefore,</div><div class='line' id='LC363'>UltiSnips issues a |:sunmap| command to remove each Select mode mapping for</div><div class='line' id='LC364'>printable characters. No other mappings are touched. In particular, UltiSnips</div><div class='line' id='LC365'>does not change existing normal, insert or visual mode mappings.</div><div class='line' id='LC366'><br/></div><div class='line' id='LC367'>If this behavior is not desired, you can disable it by adding this line to</div><div class='line' id='LC368'>your vimrc file. &gt;</div><div class='line' id='LC369'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsRemoveSelectModeMappings = 0</div><div class='line' id='LC370'><br/></div><div class='line' id='LC371'>If you want to disable this feature for specific mappings only, add them to</div><div class='line' id='LC372'>the list of mappings to be ignored. For example, the following lines in your</div><div class='line' id='LC373'>vimrc file will unmap all Select mode mappings except those mappings</div><div class='line' id='LC374'>containing either the string &quot;somePlugin&quot; or the string &quot;otherPlugin&quot; in its</div><div class='line' id='LC375'>complete definition as listed by the |:smap| command. &gt;</div><div class='line' id='LC376'><br/></div><div class='line' id='LC377'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsRemoveSelectModeMappings = 1</div><div class='line' id='LC378'>&nbsp;&nbsp;&nbsp;let g:UltiSnipsMappingsToIgnore = [ &quot;somePlugin&quot;, &quot;otherPlugin&quot; ]</div><div class='line' id='LC379'><br/></div><div class='line' id='LC380'><br/></div><div class='line' id='LC381'>3.5 Functions                                           *UltiSnips-functions*</div><div class='line' id='LC382'>-------------</div><div class='line' id='LC383'><br/></div><div class='line' id='LC384'>UltiSnips provides some functions for extending core functionality.</div><div class='line' id='LC385'><br/></div><div class='line' id='LC386'><br/></div><div class='line' id='LC387'>&nbsp;3.5.1 UltiSnips_AddSnippet                            *UltiSnips_AddSnippet*</div><div class='line' id='LC388'><br/></div><div class='line' id='LC389'>The first function is UltiSnips_AddSnippet(trigger, value, description,</div><div class='line' id='LC390'>options, ...). It adds a new snippet with the provided trigger, value,</div><div class='line' id='LC391'>description, and options to the current list of snippets. See</div><div class='line' id='LC392'>|UltiSnips-syntax| for details on the meaning of the function arguments. All</div><div class='line' id='LC393'>arguments are strings. An optional fifth argument can be used to specify the</div><div class='line' id='LC394'>filetype for the new snippet.</div><div class='line' id='LC395'><br/></div><div class='line' id='LC396'><br/></div><div class='line' id='LC397'>&nbsp;3.5.2 UltiSnips_Anon                                        *UltiSnips_Anon*</div><div class='line' id='LC398'><br/></div><div class='line' id='LC399'>The second function is UltiSnips_Anon(value, ...). It expands an anonymous</div><div class='line' id='LC400'>snippet. Anonymous snippets are defined on the spot, expanded and immediately</div><div class='line' id='LC401'>discarded again. Anonymous snippets are not added to the global list of</div><div class='line' id='LC402'>snippets, so they cannot be expanded a second time unless the function is</div><div class='line' id='LC403'>called again. The function takes three optional arguments, in order: trigger,</div><div class='line' id='LC404'>description, options. Arguments coincide with the arguments of the</div><div class='line' id='LC405'>|UltiSnips_AddSnippet| function of the same name. The trigger and options</div><div class='line' id='LC406'>arguments can change the way the snippet expands. The description is unused at</div><div class='line' id='LC407'>this point.</div><div class='line' id='LC408'><br/></div><div class='line' id='LC409'>An example use case might be this line from a reStructuredText plugin file:</div><div class='line' id='LC410'><br/></div><div class='line' id='LC411'>&nbsp;&nbsp;&nbsp;inoremap &lt;silent&gt; $$ $$&lt;C-R&gt;=UltiSnips_Anon(&#39;:latex:\`$1\`&#39;, &#39;$$&#39;)&lt;cr&gt;</div><div class='line' id='LC412'><br/></div><div class='line' id='LC413'>This expands the snippet whenever two $ signs are typed.</div><div class='line' id='LC414'>Note: The right-hand side of the mapping starts with an immediate retype of</div><div class='line' id='LC415'>the &#39;$$&#39; trigger and passes &#39;$$&#39; to the function as the trigger argument.</div><div class='line' id='LC416'>This is required in order for UltiSnips to have access to the characters</div><div class='line' id='LC417'>typed so it can determine if the trigger matches or not.</div><div class='line' id='LC418'><br/></div><div class='line' id='LC419'>&nbsp;3.5.3 UltiSnips_SnippetsInCurrentScope    *UltiSnips_SnippetsInCurrentScope*</div><div class='line' id='LC420'><br/></div><div class='line' id='LC421'>A third function is UltiSnips_SnippetsInCurrentScope which is the equivalent</div><div class='line' id='LC422'>of snipmate GetSnipsInCurrentScope function.</div><div class='line' id='LC423'>This function simply returns a vim dictionary with the snippets whose trigger</div><div class='line' id='LC424'>matches the current word.</div><div class='line' id='LC425'>This function does not add any new functionality to ultisnips directly but</div><div class='line' id='LC426'>allows to use third party plugins to integrate the current available snippets.</div><div class='line' id='LC427'><br/></div><div class='line' id='LC428'>An example of such third party plugin is SnippetCompleteSnipMate which uses</div><div class='line' id='LC429'>the function GetSnipsInCurrentScope to integrate the current available</div><div class='line' id='LC430'>snippets with user defined abbreviations and provides these and a completion</div><div class='line' id='LC431'>menu.</div><div class='line' id='LC432'>This script is located in</div><div class='line' id='LC433'>http://www.vim.org/scripts/script.php?script_id=4276.</div><div class='line' id='LC434'>Note: If you check the above website it lists two dependencies: the</div><div class='line' id='LC435'>SnippetComplete plugin and snipmate.</div><div class='line' id='LC436'>You do need the SnippetComplete plugin but you obviously don&#39;t need snipmate,</div><div class='line' id='LC437'>you just have to define the function GetSnipsInCurrentScope. Put the following</div><div class='line' id='LC438'>in your vimrc:</div><div class='line' id='LC439'><br/></div><div class='line' id='LC440'>function! GetSnipsInCurrentScope()</div><div class='line' id='LC441'>&nbsp;&nbsp;return UltiSnips_SnippetsInCurrentScope()</div><div class='line' id='LC442'>endfunction</div><div class='line' id='LC443'><br/></div><div class='line' id='LC444'><br/></div><div class='line' id='LC445'>As a second example on how to use this function consider the following</div><div class='line' id='LC446'>function and mapping definition:</div><div class='line' id='LC447'><br/></div><div class='line' id='LC448'>function! ExpandPossibleShorterSnippet()</div><div class='line' id='LC449'>&nbsp;&nbsp;if len(UltiSnips_SnippetsInCurrentScope()) == 1 &quot;only one candidate...</div><div class='line' id='LC450'>&nbsp;&nbsp;&nbsp;&nbsp;let curr_key = keys(UltiSnips_SnippetsInCurrentScope())[0]</div><div class='line' id='LC451'>&nbsp;&nbsp;&nbsp;&nbsp;normal diw</div><div class='line' id='LC452'>&nbsp;&nbsp;&nbsp;&nbsp;exe &quot;normal a&quot; . curr_key</div><div class='line' id='LC453'>&nbsp;&nbsp;&nbsp;&nbsp;exe &quot;normal a &quot;</div><div class='line' id='LC454'>&nbsp;&nbsp;&nbsp;&nbsp;return 1</div><div class='line' id='LC455'>&nbsp;&nbsp;endif</div><div class='line' id='LC456'>&nbsp;&nbsp;return 0</div><div class='line' id='LC457'>endfunction</div><div class='line' id='LC458'>inoremap &lt;silent&gt; &lt;C-L&gt; &lt;C-R&gt;=(ExpandPossibleShorterSnippet() == 0? &#39;&#39;: UltiSnips_ExpandSnippet())&lt;CR&gt;</div><div class='line' id='LC459'><br/></div><div class='line' id='LC460'>If the trigger for your snippet is lorem, you type lor, and you have no other</div><div class='line' id='LC461'>snippets whose trigger matches lor then hitting &lt;C-L&gt; will expand to whatever</div><div class='line' id='LC462'>lorem expands to.</div><div class='line' id='LC463'><br/></div><div class='line' id='LC464'><br/></div><div class='line' id='LC465'>3.6 Warning about missing python support           *UltiSnips-python-warning*</div><div class='line' id='LC466'>----------------------------------------</div><div class='line' id='LC467'><br/></div><div class='line' id='LC468'>When UltiSnips is loaded, it will check that the running Vim was compiled with</div><div class='line' id='LC469'>python support.  If no support is detected, a warning will be displayed and</div><div class='line' id='LC470'>loading of UltiSnips will be skipped.</div><div class='line' id='LC471'><br/></div><div class='line' id='LC472'>If you would like to suppress this warning message, you may add the following</div><div class='line' id='LC473'>line to your vimrc file.</div><div class='line' id='LC474'><br/></div><div class='line' id='LC475'>&nbsp;&nbsp;&nbsp;&nbsp;let g:UltiSnipsNoPythonWarning = 1</div><div class='line' id='LC476'><br/></div><div class='line' id='LC477'>This may be useful if your Vim configuration files are shared across several</div><div class='line' id='LC478'>systems where some of them may not have Vim compiled with python support.</div><div class='line' id='LC479'><br/></div><div class='line' id='LC480'>=============================================================================</div><div class='line' id='LC481'>4. Syntax                                                  *UltiSnips-syntax*</div><div class='line' id='LC482'><br/></div><div class='line' id='LC483'>This chapter describes how to write your own snippets and snippet definition</div><div class='line' id='LC484'>syntax. Examples are used to help illustrate.</div><div class='line' id='LC485'><br/></div><div class='line' id='LC486'><br/></div><div class='line' id='LC487'>4.1 Adding Snippets                               *UltiSnips-adding-snippets*</div><div class='line' id='LC488'>-------------------</div><div class='line' id='LC489'><br/></div><div class='line' id='LC490'>See |UltiSnips-snippet-search-path| for an explanation of where directories</div><div class='line' id='LC491'>with snippet definitions should be located.</div><div class='line' id='LC492'><br/></div><div class='line' id='LC493'>Using a strategy similar to how Vim detects |ftplugins|, UltiSnips iterates</div><div class='line' id='LC494'>over the snippet definition directories looking for files with names of the</div><div class='line' id='LC495'>following patterns: ft.snippets, ft_*.snippets, or ft/*, where &quot;ft&quot; is the</div><div class='line' id='LC496'>&#39;filetype&#39; of the current document and &quot;*&quot; is a shell-like wildcard matching</div><div class='line' id='LC497'>any string including the empty string. The following table shows some typical</div><div class='line' id='LC498'>snippet filenames and their associated filetype.</div><div class='line' id='LC499'><br/></div><div class='line' id='LC500'>&nbsp;&nbsp;&nbsp;&nbsp;snippet filename         filetype ~</div><div class='line' id='LC501'>&nbsp;&nbsp;&nbsp;&nbsp;ruby.snippets            ruby</div><div class='line' id='LC502'>&nbsp;&nbsp;&nbsp;&nbsp;perl.snippets            perl</div><div class='line' id='LC503'>&nbsp;&nbsp;&nbsp;&nbsp;c.snippets               c</div><div class='line' id='LC504'>&nbsp;&nbsp;&nbsp;&nbsp;c_my.snippets            c</div><div class='line' id='LC505'>&nbsp;&nbsp;&nbsp;&nbsp;c/a                      c</div><div class='line' id='LC506'>&nbsp;&nbsp;&nbsp;&nbsp;c/b.snippets             c</div><div class='line' id='LC507'>&nbsp;&nbsp;&nbsp;&nbsp;all.snippets             *all</div><div class='line' id='LC508'>&nbsp;&nbsp;&nbsp;&nbsp;all/a.snippets           *all</div><div class='line' id='LC509'><br/></div><div class='line' id='LC510'>* The &#39;all&#39; filetype is unique. It represents snippets available for use when</div><div class='line' id='LC511'>editing any document regardless of the filetype. A date insertion snippet, for</div><div class='line' id='LC512'>example, would fit well in the all.snippets file.</div><div class='line' id='LC513'><br/></div><div class='line' id='LC514'>UltiSnips understands Vim&#39;s dotted filetype syntax. For example, if you define</div><div class='line' id='LC515'>a dotted filetype for the CUDA C++ framework, e.g. &quot;:set ft=cuda.cpp&quot;, then</div><div class='line' id='LC516'>UltiSnips will search for and activate snippets for both the cuda and cpp</div><div class='line' id='LC517'>filetypes.</div><div class='line' id='LC518'><br/></div><div class='line' id='LC519'>UltiSnips doesn&#39;t understand multiple filetypes in snippet filenames, so a</div><div class='line' id='LC520'>snippet file named &quot;cuda.cpp.snippets&quot; won&#39;t match the &quot;cuda&quot; or &quot;cpp&quot;</div><div class='line' id='LC521'>filetypes, which means it won&#39;t be activated for files with the dotted</div><div class='line' id='LC522'>filetype &quot;cuda.cpp&quot; either.</div><div class='line' id='LC523'><br/></div><div class='line' id='LC524'>The extends directive provides an alternative way of combining snippet files.</div><div class='line' id='LC525'>When the extends directive is included in a snippet file, it instructs</div><div class='line' id='LC526'>UltiSnips to include all snippets from the indicated filetypes.</div><div class='line' id='LC527'><br/></div><div class='line' id='LC528'>The syntax looks like this: &gt;</div><div class='line' id='LC529'>&nbsp;&nbsp;&nbsp;extends ft1, ft2, ft3</div><div class='line' id='LC530'><br/></div><div class='line' id='LC531'>For example, the first line in cpp.snippets looks like this: &gt;</div><div class='line' id='LC532'>&nbsp;&nbsp;&nbsp;extends c</div><div class='line' id='LC533'>When UltiSnips activates snippets for a cpp file, it first looks for all c</div><div class='line' id='LC534'>snippets and activates them as well. This is a convenient way to create</div><div class='line' id='LC535'>specialized snippet files from more general ones. Multiple &quot;extends&quot; lines are</div><div class='line' id='LC536'>permitted in a snippet file, and they can be included anywhere in the file.</div><div class='line' id='LC537'><br/></div><div class='line' id='LC538'>The snippets file syntax is simple. All lines starting with a # character are</div><div class='line' id='LC539'>considered comments. Comments are ignored by UltiSnips. Use them to document</div><div class='line' id='LC540'>snippets.</div><div class='line' id='LC541'><br/></div><div class='line' id='LC542'>A line beginning with the keyword &#39;snippet&#39; marks the beginning of snippet</div><div class='line' id='LC543'>definition and a line starting with  the keyword &#39;endsnippet&#39; marks the end.</div><div class='line' id='LC544'>The snippet definition is placed between the lines. Here is a snippet of an</div><div class='line' id='LC545'>&#39;if&#39; statement for the Unix shell (sh) filetype.</div><div class='line' id='LC546'><br/></div><div class='line' id='LC547'>&nbsp;&nbsp;&nbsp;&nbsp;snippet if &quot;if ... then (if)&quot;</div><div class='line' id='LC548'>&nbsp;&nbsp;&nbsp;&nbsp;if ${2:[[ ${1:condition} ]]}; then</div><div class='line' id='LC549'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;${0:#statements}</div><div class='line' id='LC550'>&nbsp;&nbsp;&nbsp;&nbsp;fi</div><div class='line' id='LC551'>&nbsp;&nbsp;&nbsp;&nbsp;endsnippet</div><div class='line' id='LC552'><br/></div><div class='line' id='LC553'>The start line takes the following form: &gt;</div><div class='line' id='LC554'><br/></div><div class='line' id='LC555'>&nbsp;&nbsp;&nbsp;snippet tab_trigger [ &quot;description&quot; [ options ] ]</div><div class='line' id='LC556'><br/></div><div class='line' id='LC557'>The tab_trigger is required, but the description and options are optional.</div><div class='line' id='LC558'><br/></div><div class='line' id='LC559'>The &#39;tab_trigger&#39; is the word or string sequence used to trigger the snippet.</div><div class='line' id='LC560'>Generally a single word is used but the tab_trigger can include spaces. If you</div><div class='line' id='LC561'>wish to include spaces, you must wrap the tab trigger in quotes. &gt;</div><div class='line' id='LC562'><br/></div><div class='line' id='LC563'>&nbsp;&nbsp;&nbsp;&nbsp;snippet &quot;tab trigger&quot; [ &quot;description&quot; [ options ] ]</div><div class='line' id='LC564'><br/></div><div class='line' id='LC565'>The quotes are not part of the trigger. To activate the snippet type: tab trigger</div><div class='line' id='LC566'>followed by the snippet expand character.</div><div class='line' id='LC567'><br/></div><div class='line' id='LC568'>It is not technically necessary to use quotes to wrap a trigger with spaces.</div><div class='line' id='LC569'>Any matching characters will do. For example, this is a valid snippet starting</div><div class='line' id='LC570'>line. &gt;</div><div class='line' id='LC571'>&nbsp;&nbsp;&nbsp;&nbsp;snippet !tab trigger! [ &quot;description&quot; [ options ] ]</div><div class='line' id='LC572'><br/></div><div class='line' id='LC573'>Quotes can be included as part of the trigger by wrapping the trigger in</div><div class='line' id='LC574'>another character. &gt;</div><div class='line' id='LC575'>&nbsp;&nbsp;&nbsp;&nbsp;snippet !&quot;tab trigger&quot;! [ &quot;description&quot; [ options ] ]</div><div class='line' id='LC576'><br/></div><div class='line' id='LC577'>To activate this snippet one would type: &quot;tab trigger&quot;</div><div class='line' id='LC578'><br/></div><div class='line' id='LC579'>The &#39;description&#39; is a string describing the trigger. It is helpful for</div><div class='line' id='LC580'>documenting the snippet and for distinguishing it from other snippets with the</div><div class='line' id='LC581'>same tab trigger. When a snippet is activated and more than one tab trigger</div><div class='line' id='LC582'>match, UltiSnips displays a list of the matching snippets with their</div><div class='line' id='LC583'>descriptions. The user then selects the snippet they want.</div><div class='line' id='LC584'><br/></div><div class='line' id='LC585'>The &#39;options&#39; control the behavior of the snippet. Options are indicated by</div><div class='line' id='LC586'>single characters. The &#39;options&#39; characters for a snippet are combined into</div><div class='line' id='LC587'>a word without spaces.</div><div class='line' id='LC588'><br/></div><div class='line' id='LC589'>The options currently supported are: &gt;</div><div class='line' id='LC590'>&nbsp;&nbsp;&nbsp;!   Overwrite - A snippet with this option will overwrite all previously</div><div class='line' id='LC591'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;defined snippets with an identical tab trigger. What previously means</div><div class='line' id='LC592'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;in this context depends on the order UltiSnips traverses the</div><div class='line' id='LC593'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#39;runtimepath&#39;. (Compare g:UltiSnipsDontReverseSearchPath). This might</div><div class='line' id='LC594'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;depend on your installation type (Pathogen, Bzr, Vundle). The default</div><div class='line' id='LC595'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;behavior is to display list of snippets matching the tab trigger and</div><div class='line' id='LC596'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;let the user pick the one they want. Use this option to overwrite</div><div class='line' id='LC597'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;bundled snippets with user defined ones.</div><div class='line' id='LC598'><br/></div><div class='line' id='LC599'>&nbsp;&nbsp;&nbsp;b   Beginning of line - A snippet with this option is expanded only if the</div><div class='line' id='LC600'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;tab trigger is the first word on the line. In other words, if only</div><div class='line' id='LC601'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;whitespace precedes the tab trigger, expand. The default is to expand</div><div class='line' id='LC602'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;snippets at any position regardless of the preceding non-whitespace</div><div class='line' id='LC603'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;characters.</div><div class='line' id='LC604'><br/></div><div class='line' id='LC605'>&nbsp;&nbsp;&nbsp;i   In-word expansion - By default a snippet is expanded only if the tab</div><div class='line' id='LC606'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;trigger is the first word on the line or is preceded by one or more</div><div class='line' id='LC607'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;whitespace characters. A snippet with this option is expanded</div><div class='line' id='LC608'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;regardless of the preceding character. In other words, the snippet can</div><div class='line' id='LC609'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;be triggered in the middle of a word.</div><div class='line' id='LC610'><br/></div><div class='line' id='LC611'>&nbsp;&nbsp;&nbsp;w   Word boundary - With this option, the snippet is expanded if</div><div class='line' id='LC612'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;the tab trigger start matches a word boundary and the tab trigger end</div><div class='line' id='LC613'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;matches a word boundary. In other words the tab trigger must be</div><div class='line' id='LC614'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;preceded and followed by non-word characters. Word characters are</div><div class='line' id='LC615'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;letters, digits and the underscore. Use this option, for example, to</div><div class='line' id='LC616'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;permit expansion where the tab trigger follows punctuation without</div><div class='line' id='LC617'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;expanding suffixes of larger words.</div><div class='line' id='LC618'><br/></div><div class='line' id='LC619'>&nbsp;&nbsp;&nbsp;r   Regular expression - With this option, the tab trigger is expected to</div><div class='line' id='LC620'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;be a python regular expression. The snippet is expanded if the recently</div><div class='line' id='LC621'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;typed characters match the regular expression. Note: The regular</div><div class='line' id='LC622'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;expression MUST be quoted (or surrounded with another character) like a</div><div class='line' id='LC623'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;multi-word tab trigger (see above) whether it has spaces or not. A</div><div class='line' id='LC624'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;resulting match is passed to any python code blocks in the snippet</div><div class='line' id='LC625'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;definition as the local variable &quot;match&quot;.</div><div class='line' id='LC626'><br/></div><div class='line' id='LC627'>&nbsp;&nbsp;&nbsp;t   Do not expand tabs - If a snippet definition includes leading tab</div><div class='line' id='LC628'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;characters, by default UltiSnips expands the tab characters honoring</div><div class='line' id='LC629'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;the Vim &#39;shiftwidth&#39;, &#39;softtabstop&#39;, &#39;expandtab&#39; and &#39;tabstop&#39;</div><div class='line' id='LC630'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;indentation settings. (For example, if &#39;expandtab&#39; is set, the tab is</div><div class='line' id='LC631'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;replaced with spaces.) If this option is set, UltiSnips will ignore the</div><div class='line' id='LC632'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Vim settings and insert the tab characters as is. This option is useful</div><div class='line' id='LC633'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;for snippets involved with tab delimited formats, for example.</div><div class='line' id='LC634'><br/></div><div class='line' id='LC635'>&nbsp;&nbsp;&nbsp;s   Remove whitespace immediately before the cursor at the end of a line</div><div class='line' id='LC636'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;before jumping to the next tabstop.  This is useful if there is a</div><div class='line' id='LC637'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;tabstop with optional text at the end of a line.</div><div class='line' id='LC638'><br/></div><div class='line' id='LC639'>The end line is the &#39;endsnippet&#39; keyword on a line by itself. &gt;</div><div class='line' id='LC640'><br/></div><div class='line' id='LC641'>&nbsp;&nbsp;&nbsp;endsnippet</div><div class='line' id='LC642'><br/></div><div class='line' id='LC643'>When parsing snippet files, UltiSnips chops the trailing newline character</div><div class='line' id='LC644'>from the &#39;endsnippet&#39; end line.</div><div class='line' id='LC645'><br/></div><div class='line' id='LC646'><br/></div><div class='line' id='LC647'>&nbsp;4.1.1 Character Escaping:                     *UltiSnips-character-escaping*</div><div class='line' id='LC648'><br/></div><div class='line' id='LC649'>In snippet definitions, the characters &#39;`&#39;, &#39;{&#39;, &#39;$&#39;, &#39;\&#39; and &quot;&#39;&quot; (single</div><div class='line' id='LC650'>quote) have special meaning. If you want to insert one of these characters</div><div class='line' id='LC651'>literally, escape them with a backslash, &#39;\&#39;.</div><div class='line' id='LC652'><br/></div><div class='line' id='LC653'><br/></div><div class='line' id='LC654'>4.2 Plaintext Snippets                         *UltiSnips-plaintext-snippets*</div><div class='line' id='LC655'>----------------------</div><div class='line' id='LC656'><br/></div><div class='line' id='LC657'>To illustrate plaintext snippets, let&#39;s begin with a simple example. You can</div><div class='line' id='LC658'>try the examples yourself. Simply edit a new file with Vim. Example snippets</div><div class='line' id='LC659'>will be added to the &#39;all.snippets&#39; file, so you&#39;ll want to open it in Vim for</div><div class='line' id='LC660'>editing as well. &gt;</div><div class='line' id='LC661'>&nbsp;&nbsp;&nbsp;~/.vim/UltiSnips/all.snippets</div><div class='line' id='LC662'><br/></div><div class='line' id='LC663'>Add this snippet to &#39;all.snippets&#39; and save the file.</div><div class='line' id='LC664'><br/></div><div class='line' id='LC665'>------------------- SNIP -------------------</div><div class='line' id='LC666'>snippet bye &quot;My mail signature&quot;</div><div class='line' id='LC667'>Good bye, Sir. Hope to talk to you soon.</div><div class='line' id='LC668'>- Arthur, King of Britain</div><div class='line' id='LC669'>endsnippet</div><div class='line' id='LC670'>------------------- SNAP -------------------</div><div class='line' id='LC671'><br/></div><div class='line' id='LC672'>UltiSnips detects when you write changes to a snippets file and automatically</div><div class='line' id='LC673'>makes the changes active. So in the empty buffer, type the tab trigger &#39;bye&#39;</div><div class='line' id='LC674'>and then press the &lt;Tab&gt; key.</div><div class='line' id='LC675'><br/></div><div class='line' id='LC676'>bye&lt;Tab&gt; --&gt;</div><div class='line' id='LC677'>Good bye, Sir. Hope to talk to you soon.</div><div class='line' id='LC678'>- Arthur, King of Britain</div><div class='line' id='LC679'><br/></div><div class='line' id='LC680'>The word &#39;bye&#39; will be replaced with the text of the snippet definition.</div><div class='line' id='LC681'><br/></div><div class='line' id='LC682'><br/></div><div class='line' id='LC683'>4.3 Visual Placeholder                         *UltiSnips-visual-placeholder*</div><div class='line' id='LC684'>----------------------</div><div class='line' id='LC685'><br/></div><div class='line' id='LC686'>Snippets can contain a special placeholder called ${VISUAL}. The ${VISUAL}</div><div class='line' id='LC687'>variable is expanded with the text selected just prior to expanding the</div><div class='line' id='LC688'>snippet.</div><div class='line' id='LC689'><br/></div><div class='line' id='LC690'>To see how a snippet with a ${VISUAL} placeholder works, define a snippet with</div><div class='line' id='LC691'>the placeholder, use Vim&#39;s Visual mode to select some text, and then press the</div><div class='line' id='LC692'>key you use to trigger expanding a snippet (see g:UltiSnipsExpandTrigger). The</div><div class='line' id='LC693'>selected text is deleted, and you are dropped into Insert mode. Now type the</div><div class='line' id='LC694'>snippet tab trigger and press the key to trigger expansion. As the snippet</div><div class='line' id='LC695'>expands, the previously selected text is printed in place of the ${VISUAL}</div><div class='line' id='LC696'>placeholder.</div><div class='line' id='LC697'><br/></div><div class='line' id='LC698'>The ${VISUAL} placeholder can contain default text to use when the snippet has</div><div class='line' id='LC699'>been triggered when not in Visual mode. The syntax is: &gt;</div><div class='line' id='LC700'>&nbsp;&nbsp;&nbsp;&nbsp;${VISUAL:default text}</div><div class='line' id='LC701'><br/></div><div class='line' id='LC702'>The ${VISUAL} placeholder can also define a transformation (see</div><div class='line' id='LC703'>|UltiSnips-transformations|). The syntax is: &gt;</div><div class='line' id='LC704'>&nbsp;&nbsp;&nbsp;&nbsp;${VISUAL:default/search/replace/option}.</div><div class='line' id='LC705'><br/></div><div class='line' id='LC706'>Here is a simple example illustrating a visual transformation. The snippet</div><div class='line' id='LC707'>will take selected text, replace every instance of &quot;should&quot; within it with</div><div class='line' id='LC708'>&quot;is&quot; , and wrap the result in tags.</div><div class='line' id='LC709'><br/></div><div class='line' id='LC710'>------------------- SNIP -------------------</div><div class='line' id='LC711'>snippet t</div><div class='line' id='LC712'>&lt;tag&gt;${VISUAL:inside text/should/is/g}&lt;/tag&gt;</div><div class='line' id='LC713'>endsnippet</div><div class='line' id='LC714'>------------------- SNAP -------------------</div><div class='line' id='LC715'><br/></div><div class='line' id='LC716'>Start with this line of text: &gt;</div><div class='line' id='LC717'>&nbsp;&nbsp;&nbsp;this should be cool</div><div class='line' id='LC718'><br/></div><div class='line' id='LC719'>Position the cursor on the word &quot;should&quot;, then press the key sequence: viw</div><div class='line' id='LC720'>(visual mode -&gt; select inner word). Then press &lt;Tab&gt;, type &quot;t&quot; and press &lt;Tab&gt;</div><div class='line' id='LC721'>again. The result is: &gt;</div><div class='line' id='LC722'>&nbsp;&nbsp;&nbsp;-&gt; this &lt;tag&gt;is&lt;/tag&gt; be cool</div><div class='line' id='LC723'><br/></div><div class='line' id='LC724'>If you expand this snippet while not in Visual mode (e.g., in Insert mode type</div><div class='line' id='LC725'>t&lt;Tab&gt;), you will get: &gt;</div><div class='line' id='LC726'>&nbsp;&nbsp;&nbsp;&lt;tag&gt;inside text&lt;/tag&gt;</div><div class='line' id='LC727'><br/></div><div class='line' id='LC728'><br/></div><div class='line' id='LC729'>4.4 Interpolation                                   *UltiSnips-interpolation*</div><div class='line' id='LC730'>-----------------</div><div class='line' id='LC731'><br/></div><div class='line' id='LC732'>&nbsp;4.4.1 Shellcode:                                       *UltiSnips-shellcode*</div><div class='line' id='LC733'><br/></div><div class='line' id='LC734'>Snippets can include shellcode. Put a shell command in a snippet and when the</div><div class='line' id='LC735'>snippet is expanded, the shell command is replaced by the output produced when</div><div class='line' id='LC736'>the command is executed. The syntax for shellcode is simple: wrap the code in</div><div class='line' id='LC737'>backticks, &#39;`&#39;. When a snippet is expanded, UltiSnips runs shellcode by first</div><div class='line' id='LC738'>writing it to a temporary script and then executing the script. The shellcode</div><div class='line' id='LC739'>is replaced by the standard output. Anything you can run as a script can be</div><div class='line' id='LC740'>used in shellcode. Include a shebang line, for example, #!/usr/bin/perl, and</div><div class='line' id='LC741'>your snippet has the ability to run scripts using other programs, perl, for</div><div class='line' id='LC742'>example.</div><div class='line' id='LC743'><br/></div><div class='line' id='LC744'>Here are some examples. This snippet uses a shell command to insert the</div><div class='line' id='LC745'>current date.</div><div class='line' id='LC746'><br/></div><div class='line' id='LC747'>------------------- SNIP -------------------</div><div class='line' id='LC748'>snippet today</div><div class='line' id='LC749'>Today is the `date +%d.%m.%y`.</div><div class='line' id='LC750'>endsnippet</div><div class='line' id='LC751'>------------------- SNAP -------------------</div><div class='line' id='LC752'><br/></div><div class='line' id='LC753'>today&lt;tab&gt; -&gt;</div><div class='line' id='LC754'>Today is the 15.07.09.</div><div class='line' id='LC755'><br/></div><div class='line' id='LC756'><br/></div><div class='line' id='LC757'>This example inserts the current date using perl.</div><div class='line' id='LC758'><br/></div><div class='line' id='LC759'>------------------- SNIP -------------------</div><div class='line' id='LC760'>snippet today</div><div class='line' id='LC761'>Today is `#!/usr/bin/perl</div><div class='line' id='LC762'>@a = localtime(); print $a[3] . &#39;.&#39; . $a[4] . &#39;.&#39; . ($a[5]+1900);`.</div><div class='line' id='LC763'>endsnippet</div><div class='line' id='LC764'>------------------- SNAP -------------------</div><div class='line' id='LC765'>today&lt;tab&gt; -&gt;</div><div class='line' id='LC766'>Today is 15.6.2009.</div><div class='line' id='LC767'><br/></div><div class='line' id='LC768'><br/></div><div class='line' id='LC769'>&nbsp;4.4.2 VimScript:                                       *UltiSnips-vimscript*</div><div class='line' id='LC770'><br/></div><div class='line' id='LC771'>You can also use Vim scripts (sometimes called VimL) in interpolation. The</div><div class='line' id='LC772'>syntax is similar to shellcode. Wrap the code in backticks and to distinguish</div><div class='line' id='LC773'>it as a Vim script, start the code with &#39;!v&#39;. Here is an example that counts</div><div class='line' id='LC774'>the indent of the current line:</div><div class='line' id='LC775'><br/></div><div class='line' id='LC776'>------------------- SNIP -------------------</div><div class='line' id='LC777'>snippet indent</div><div class='line' id='LC778'>Indent is: `!v indent(&quot;.&quot;)`.</div><div class='line' id='LC779'>endsnippet</div><div class='line' id='LC780'>------------------- SNAP -------------------</div><div class='line' id='LC781'>&nbsp;&nbsp;&nbsp;&nbsp;(note the 4 spaces in front): indent&lt;tab&gt; -&gt;</div><div class='line' id='LC782'>&nbsp;&nbsp;&nbsp;&nbsp;(note the 4 spaces in front): Indent is: 4.</div><div class='line' id='LC783'><br/></div><div class='line' id='LC784'><br/></div><div class='line' id='LC785'>&nbsp;4.4.3 Python:                                             *UltiSnips-python*</div><div class='line' id='LC786'><br/></div><div class='line' id='LC787'>Python interpolation is by far the most powerful. The syntax is similar to Vim</div><div class='line' id='LC788'>scripts except code is started with &#39;!p&#39;. Python scripts can be run using the</div><div class='line' id='LC789'>python shebang &#39;#!/usr/bin/python&#39;, but using the &#39;!p&#39; format comes with some</div><div class='line' id='LC790'>predefined objects and variables, which can simplify and shorten code. For</div><div class='line' id='LC791'>example, a &#39;snip&#39; object instance is implied in python code. Python code using</div><div class='line' id='LC792'>the &#39;!p&#39; indicator differs in another way. Generally when a snippet is</div><div class='line' id='LC793'>expanded the standard output of code replaces the code. With python code the</div><div class='line' id='LC794'>value of the &#39;rv&#39; property of the &#39;snip&#39; instance replaces the code. Standard</div><div class='line' id='LC795'>output is ignored.</div><div class='line' id='LC796'><br/></div><div class='line' id='LC797'>The variables automatically defined in python code are: &gt;</div><div class='line' id='LC798'><br/></div><div class='line' id='LC799'>&nbsp;&nbsp;&nbsp;fn   - The current filename</div><div class='line' id='LC800'>&nbsp;&nbsp;&nbsp;path - The complete path to the current file</div><div class='line' id='LC801'>&nbsp;&nbsp;&nbsp;t    - The values of the placeholders, t[1] is the text of ${1}, and so on</div><div class='line' id='LC802'>&nbsp;&nbsp;&nbsp;snip - UltiSnips.TextObjects.SnippetUtil object instance. Has methods that</div><div class='line' id='LC803'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;simplify indentation handling.</div><div class='line' id='LC804'><br/></div><div class='line' id='LC805'>The &#39;snip&#39; object provides the following methods: &gt;</div><div class='line' id='LC806'><br/></div><div class='line' id='LC807'>&nbsp;&nbsp;&nbsp;&nbsp;snip.mkline(line=&quot;&quot;, indent=None):</div><div class='line' id='LC808'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Returns a line ready to be appended to the result. If indent</div><div class='line' id='LC809'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;is None, then mkline prepends spaces and/or tabs appropriate to the</div><div class='line' id='LC810'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;current &#39;tabstop&#39; and &#39;expandtab&#39; variables.</div><div class='line' id='LC811'><br/></div><div class='line' id='LC812'>&nbsp;&nbsp;&nbsp;&nbsp;snip.shift(amount=1):</div><div class='line' id='LC813'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Shifts the default indentation level used by mkline right by the</div><div class='line' id='LC814'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;number of spaces defined by &#39;shiftwidth&#39;, &#39;amount&#39; times.</div><div class='line' id='LC815'><br/></div><div class='line' id='LC816'>&nbsp;&nbsp;&nbsp;&nbsp;snip.unshift(amount=1):</div><div class='line' id='LC817'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Shifts the default indentation level used by mkline left by the</div><div class='line' id='LC818'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;number of spaces defined by &#39;shiftwidth&#39;, &#39;amount&#39; times.</div><div class='line' id='LC819'><br/></div><div class='line' id='LC820'>&nbsp;&nbsp;&nbsp;&nbsp;snip.reset_indent():</div><div class='line' id='LC821'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Resets the indentation level to its initial value.</div><div class='line' id='LC822'><br/></div><div class='line' id='LC823'>&nbsp;&nbsp;&nbsp;&nbsp;snip.opt(var, default):</div><div class='line' id='LC824'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Checks if the Vim variable &#39;var&#39; has been set. If so, it returns the</div><div class='line' id='LC825'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;variable&#39;s value; otherwise, it returns the value of &#39;default&#39;.</div><div class='line' id='LC826'><br/></div><div class='line' id='LC827'>The &#39;snip&#39; object provides some properties as well: &gt;</div><div class='line' id='LC828'><br/></div><div class='line' id='LC829'>&nbsp;&nbsp;&nbsp;&nbsp;snip.rv:</div><div class='line' id='LC830'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#39;rv&#39; is the return value, the text that will replace the python block</div><div class='line' id='LC831'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;in the snippet definition. It is initialized to the empty string. This</div><div class='line' id='LC832'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;deprecates the &#39;res&#39; variable.</div><div class='line' id='LC833'><br/></div><div class='line' id='LC834'>&nbsp;&nbsp;&nbsp;&nbsp;snip.c:</div><div class='line' id='LC835'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The text currently in the python block&#39;s position within the snippet.</div><div class='line' id='LC836'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;It is set to empty string as soon as interpolation is completed. Thus</div><div class='line' id='LC837'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;you can check if snip.c is != &quot;&quot; to make sure that the interpolation</div><div class='line' id='LC838'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;is only done once. This deprecates the &quot;cur&quot; variable.</div><div class='line' id='LC839'><br/></div><div class='line' id='LC840'>&nbsp;&nbsp;&nbsp;&nbsp;snip.v:</div><div class='line' id='LC841'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Data related to the ${VISUAL} placeholder. Using regular</div><div class='line' id='LC907'>expressions in this manner has some drawbacks:</div><div class='line' id='LC908'>1. If you use the &lt;Tab&gt; key for both expanding snippets and completion then</div><div class='line' id='LC909'>&nbsp;&nbsp;&nbsp;if you typed &quot;be form&lt;Tab&gt;&quot; expecting the completion &quot;be formatted&quot;, you</div><div class='line' id='LC910'>&nbsp;&nbsp;&nbsp;would end up with the above SNAP instead, not what you want.</div><div class='line' id='LC911'>2. The snippet is harder to read.</div><div class='line' id='LC912'><br/></div><div class='line' id='LC913'><br/></div><div class='line' id='LC914'>&nbsp;4.4.4 Global Snippets:                                   *UltiSnips-globals*</div><div class='line' id='LC915'><br/></div><div class='line' id='LC916'>Global snippets provide a way to reuse common code in multiple snippets.</div><div class='line' id='LC917'>Currently, only python code is supported. The result of executing the contents</div><div class='line' id='LC918'>of a global snippet is put into the globals of each python block in the</div><div class='line' id='LC919'>snippet file. To create a global snippet, use the keyword &#39;global&#39; in place of</div><div class='line' id='LC920'>&#39;snippet&#39;, and for python code, you use &#39;!p&#39; for the trigger. For example, the</div><div class='line' id='LC921'>following snippet produces the same output as the last example . However, with</div><div class='line' id='LC922'>this syntax the &#39;upper_right&#39; snippet can be reused by other snippets.</div><div class='line' id='LC923'><br/></div><div class='line' id='LC924'>------------------- SNIP -------------------</div><div class='line' id='LC925'>global !p</div><div class='line' id='LC926'>def upper_right(inp):</div><div class='line' id='LC927'>&nbsp;&nbsp;&nbsp;&nbsp;return (75 - 2 * len(inp))*&#39; &#39; + inp.upper()</div><div class='line' id='LC928'>endglobal</div><div class='line' id='LC929'><br/></div><div class='line' id='LC930'>snippet wow</div><div class='line' id='LC931'>${1:Text}`!p snip.rv = upper_right(t[1])`</div><div class='line' id='LC932'>endsnippet</div><div class='line' id='LC933'>------------------- SNAP -------------------</div><div class='line' id='LC934'>wow&lt;tab&gt;Hello World -&gt;</div><div class='line' id='LC935'>Hello World                                                     HELLO WORLD</div><div class='line' id='LC936'><br/></div><div class='line' id='LC937'>Python global functions can be stored in a python module and then imported.</div><div class='line' id='LC938'>This makes global functions easily accessible to all snippet files.</div><div class='line' id='LC939'><br/></div><div class='line' id='LC940'>First, add the directory modules are stored in to the python search path. For</div><div class='line' id='LC941'>example, add this line to your vimrc file. &gt;</div><div class='line' id='LC942'><br/></div><div class='line' id='LC943'>&nbsp;&nbsp;&nbsp;py import sys; sys.path.append(&quot;/home/sirver/.vim/python&quot;)</div><div class='line' id='LC944'><br/></div><div class='line' id='LC945'>Now, import modules from this directory using a global snippet in your snippet</div><div class='line' id='LC946'>file &gt;</div><div class='line' id='LC947'><br/></div><div class='line' id='LC948'>&nbsp;&nbsp;&nbsp;global !p</div><div class='line' id='LC949'>&nbsp;&nbsp;&nbsp;from my_snippet_helpers import *</div><div class='line' id='LC950'>&nbsp;&nbsp;&nbsp;endglobals</div><div class='line' id='LC951'><br/></div><div class='line' id='LC952'><br/></div><div class='line' id='LC953'>4.4 Tabstops and Placeholders   *UltiSnips-tabstops* *UltiSnips-placeholders*</div><div class='line' id='LC954'>-----------------------------</div><div class='line' id='LC955'><br/></div><div class='line' id='LC956'>Snippets are used to quickly insert reused text into a document. Often the</div><div class='line' id='LC957'>text has a fixed structure with variable components. Tabstops are used to</div><div class='line' id='LC958'>simplify modifying the variable content. With tabstops you can easily place</div><div class='line' id='LC959'>the cursor at the point of the variable content, enter the content you want,</div><div class='line' id='LC960'>then jump to the next variable component, enter that content, and continue</div><div class='line' id='LC961'>until all the variable components are complete.</div><div class='line' id='LC962'><br/></div><div class='line' id='LC963'>The syntax for a tabstop is the dollar sign followed by a number, for example,</div><div class='line' id='LC964'>&#39;$1&#39;. Tabstops start at number 1 and are followed in sequential order. The</div><div class='line' id='LC965'>&#39;$0&#39; tabstop is a special tabstop. It is always the last tabstop in the</div><div class='line' id='LC966'>snippet no matter how many tabstops are defined.</div><div class='line' id='LC967'><br/></div><div class='line' id='LC968'>Here is a simple example.</div><div class='line' id='LC969'><br/></div><div class='line' id='LC970'>------------------- SNIP -------------------</div><div class='line' id='LC971'>snippet letter</div><div class='line' id='LC972'>Dear $1,</div><div class='line' id='LC973'>$0</div><div class='line' id='LC974'>Yours sincerely,</div><div class='line' id='LC975'>$2</div><div class='line' id='LC976'>endsnippet</div><div class='line' id='LC977'>------------------- SNAP -------------------</div><div class='line' id='LC978'>letter&lt;tab&gt;Ben&lt;c-j&gt;Paul&lt;c-j&gt;Thanks for suggesting UltiSnips!-&gt;</div><div class='line' id='LC979'>Dear Ben,</div><div class='line' id='LC980'>Thanks for suggesting UltiSnips!</div><div class='line' id='LC981'>Yours sincerely,</div><div class='line' id='LC982'>Paul</div><div class='line' id='LC983'><br/></div><div class='line' id='LC984'><br/></div><div class='line' id='LC985'>You can use &lt;c-j&gt; to jump to the next tabstop, and &lt;c-k&gt; to jump to the</div><div class='line' id='LC986'>previous. The &lt;Tab&gt; key was not used for jumping forward because many people</div><div class='line' id='LC987'>(myself included) use &lt;Tab&gt; for completion. See |UltiSnips-triggers| for</div><div class='line' id='LC988'>help on defining different keys for tabstops.</div><div class='line' id='LC989'><br/></div><div class='line' id='LC990'>Many times it is useful to have some default text for a tabstop. The default</div><div class='line' id='LC991'>text may be a value commonly used for the variable component, or it may be a</div><div class='line' id='LC992'>word or phrase that reminds you what is expected for the variable component.</div><div class='line' id='LC993'>To include default text, the syntax is &#39;${1:value}&#39;.</div><div class='line' id='LC994'><br/></div><div class='line' id='LC995'>The following example illustrates a snippet for the shell &#39;case&#39; statement.</div><div class='line' id='LC996'>The tabstops use default values to remind the user of what value is expected.</div><div class='line' id='LC997'><br/></div><div class='line' id='LC998'>------------------- SNIP -------------------</div><div class='line' id='LC999'>snippet case</div><div class='line' id='LC1000'>case ${1:word} in</div><div class='line' id='LC1001'>&nbsp;&nbsp;&nbsp;&nbsp;${2:pattern} ) $0;;</div><div class='line' id='LC1002'>esac</div><div class='line' id='LC1003'>endsnippet</div><div class='line' id='LC1004'>------------------- SNAP -------------------</div><div class='line' id='LC1005'><br/></div><div class='line' id='LC1006'>case&lt;tab&gt;$option&lt;c-j&gt;-v&lt;c-j&gt;verbose=true</div><div class='line' id='LC1007'>case $option in</div><div class='line' id='LC1008'>&nbsp;&nbsp;&nbsp;&nbsp;-v ) verbose=true;;</div><div class='line' id='LC1009'>esac</div><div class='line' id='LC1010'><br/></div><div class='line' id='LC1011'><br/></div><div class='line' id='LC1012'>Sometimes it is useful to have a tabstop within a tabstop. To do this, simply</div><div class='line' id='LC1013'>include the nested tabstop as part of the default text. Consider the following</div><div class='line' id='LC1014'>example illustrating an HTML anchor snippet.</div><div class='line' id='LC1015'><br/></div><div class='line' id='LC1016'><br/></div><div class='line' id='LC1017'>------------------- SNIP -------------------</div><div class='line' id='LC1018'>snippet a</div><div class='line' id='LC1019'>&lt;a href=&quot;${1:http://www.${2:example.com}}&quot;&lt;/a&gt;</div><div class='line' id='LC1020'>&nbsp;&nbsp;&nbsp;&nbsp;$0</div><div class='line' id='LC1021'>&lt;/a&gt;</div><div class='line' id='LC1022'>endsnippet</div><div class='line' id='LC1023'>------------------- SNAP -------------------</div><div class='line' id='LC1024'><br/></div><div class='line' id='LC1025'>When this snippet is expanded, the first tabstop has a default value of</div><div class='line' id='LC1026'>&#39;http://www.example.com&#39;. If you want the &#39;http://&#39; schema, jump to the next</div><div class='line' id='LC1027'>tabstop. It has a default value of &#39;example.com&#39;. This can be replaced by</div><div class='line' id='LC1028'>typing whatever domain you want.</div><div class='line' id='LC1029'><br/></div><div class='line' id='LC1030'>a&lt;tab&gt;&lt;c-j&gt;google.com&lt;c-j&gt;Google -&gt;</div><div class='line' id='LC1031'>&lt;a href=&quot;http://www.google.com&quot;&gt;</div><div class='line' id='LC1032'>&nbsp;&nbsp;&nbsp;&nbsp;Google</div><div class='line' id='LC1033'>&lt;/a&gt;</div><div class='line' id='LC1034'><br/></div><div class='line' id='LC1035'>If at the first tabstop you want a different url schema or want to replace the</div><div class='line' id='LC1036'>default url with a named anchor, &#39;#name&#39;, for example, just type the value you</div><div class='line' id='LC1037'>want.</div><div class='line' id='LC1038'><br/></div><div class='line' id='LC1039'>a&lt;tab&gt;#top&lt;c-j&gt;Top -&gt;</div><div class='line' id='LC1040'>&lt;a href=&quot;#top&quot;&gt;</div><div class='line' id='LC1041'>&nbsp;&nbsp;&nbsp;&nbsp;Top</div><div class='line' id='LC1042'>&lt;/a&gt;</div><div class='line' id='LC1043'><br/></div><div class='line' id='LC1044'>In the last example, typing any text at the first tabstop replaces the default</div><div class='line' id='LC1045'>value, including the second tabstop, with the typed text. So the second</div><div class='line' id='LC1046'>tabstop is essentially deleted. When a tabstop jump is triggered, UltiSnips</div><div class='line' id='LC1047'>moves to the next remaining tabstop &#39;$0&#39;. This feature can be used</div><div class='line' id='LC1048'>intentionally as a handy way for providing optional tabstop values to the</div><div class='line' id='LC1049'>user. Here is an example to illustrate.</div><div class='line' id='LC1050'><br/></div><div class='line' id='LC1051'>------------------- SNIP -------------------</div><div class='line' id='LC1052'>snippet a</div><div class='line' id='LC1053'>&lt;a href=&quot;$1&quot;${2: class=&quot;${3:link}&quot;}&gt;</div><div class='line' id='LC1054'>&nbsp;&nbsp;&nbsp;&nbsp;$0</div><div class='line' id='LC1055'>&lt;/a&gt;</div><div class='line' id='LC1056'>endsnippet</div><div class='line' id='LC1057'>------------------- SNAP -------------------</div><div class='line' id='LC1058'><br/></div><div class='line' id='LC1059'>Here, &#39;$1&#39; marks the first tabstop. It is assumed you always want to add a</div><div class='line' id='LC1060'>value for the &#39;href&#39; attribute. After entering the url and pressing &lt;c-j&gt;, the</div><div class='line' id='LC1061'>snippet will jump to the second tabstop, &#39;$2&#39;. This tabstop is optional. The</div><div class='line' id='LC1062'>default text is &#39; class=&quot;link&quot;&#39;. You can press &lt;c-j&gt; to accept the tabstop,</div><div class='line' id='LC1063'>and the snippet will jump to the third tabstop, &#39;$3&#39;, and you can enter the</div><div class='line' id='LC1064'>class attribute value, or, at the second tabstop you can press the backspace</div><div class='line' id='LC1065'>key thereby replacing the second tabstop default with an empty string,</div><div class='line' id='LC1066'>essentially removing it. During snippet expansion when you</div><div class='line' id='LC1088'>enter the value for a tabstop, all mirrors of that tabstop are replaced with</div><div class='line' id='LC1089'>the same value. To mirror a tabstop simply insert the tabstop again using the</div><div class='line' id='LC1090'>&quot;dollar sign followed by a number&quot; syntax, e.g., &#39;$1&#39;.</div><div class='line' id='LC1091'><br/></div><div class='line' id='LC1092'>A tabstop can be mirrored multiple times in one snippet, and more than one</div><div class='line' id='LC1093'>tabstop can be mirrored in the same snippet. A mirrored tabstop can have a</div><div class='line' id='LC1094'>default value defined. Only the first instance of the tabstop need have a</div><div class='line' id='LC1095'>default value. Mirrored tabstop will take on the default value automatically.</div><div class='line' id='LC1096'><br/></div><div class='line' id='LC1097'>Mirrors are handy for start-end tags, for example, TeX &#39;begin&#39; and &#39;end&#39; tag</div><div class='line' id='LC1098'>labels, XML and HTML tags, and C code #ifndef blocks. Here are some snippet</div><div class='line' id='LC1099'>examples.</div><div class='line' id='LC1100'><br/></div><div class='line' id='LC1101'>------------------- SNIP -------------------</div><div class='line' id='LC1102'>snippet env</div><div class='line' id='LC1103'>\begin{${1:enumerate}}</div><div class='line' id='LC1104'>&nbsp;&nbsp;&nbsp;&nbsp;$0</div><div class='line' id='LC1105'>\end{$1}</div><div class='line' id='LC1106'>endsnippet</div><div class='line' id='LC1107'>------------------- SNAP -------------------</div><div class='line' id='LC1108'>env&lt;tab&gt;itemize -&gt;</div><div class='line' id='LC1109'>\begin{itemize}</div><div class='line' id='LC1110'><br/></div><div class='line' id='LC1111'>\end{itemize}</div><div class='line' id='LC1112'><br/></div><div class='line' id='LC1113'>------------------- SNIP -------------------</div><div class='line' id='LC1114'>snippet ifndef</div><div class='line' id='LC1115'>#ifndef ${1:SOME_DEFINE}</div><div class='line' id='LC1116'>#define $1</div><div class='line' id='LC1117'>$0</div><div class='line' id='LC1118'>#endif /* $1 */</div><div class='line' id='LC1119'>endsnippet</div><div class='line' id='LC1120'>------------------- SNAP -------------------</div><div class='line' id='LC1121'>ifndef&lt;tab&gt;WIN32 -&gt;</div><div class='line' id='LC1122'>#ifndef WIN32</div><div class='line' id='LC1123'>#define WIN32</div><div class='line' id='LC1124'><br/></div><div class='line' id='LC1125'>#endif /* WIN32 */</div><div class='line' id='LC1126'><br/></div><div class='line' id='LC1127'><br/></div><div class='line' id='LC1128'>4.7 Transformations                               *UltiSnips-transformations*</div><div class='line' id='LC1129'>-------------------</div><div class='line' id='LC1130'><br/></div><div class='line' id='LC1131'>Note: Transformations are a bit difficult to grasp so this chapter is divided</div><div class='line' id='LC1132'>into two sections. The first describes transformations and their syntax, and</div><div class='line' id='LC1133'>the second illustrates transformations with demos.</div><div class='line' id='LC1134'><br/></div><div class='line' id='LC1135'>Transformations are like mirrors but instead of just copying text from the</div><div class='line' id='LC1136'>original tabstop verbatim, a regular expression is matched to the content of</div><div class='line' id='LC1137'>the referenced tabstop and a transformation is then applied to the matched</div><div class='line' id='LC1138'>pattern. The syntax and functionality of transformations in UltiSnips follow</div><div class='line' id='LC1139'>very closely to TextMate transformations.</div><div class='line' id='LC1140'><br/></div><div class='line' id='LC1141'>A transformation has the following syntax: &gt;</div><div class='line' id='LC1142'>&nbsp;&nbsp;&nbsp;${&lt;tab_stop_no/regular_expression/replacement/options}</div><div class='line' id='LC1143'><br/></div><div class='line' id='LC1144'>The components are defined as follows: &gt;</div><div class='line' id='LC1145'>&nbsp;&nbsp;&nbsp;tab_stop_no        - The number of the tabstop to reference</div><div class='line' id='LC1146'>&nbsp;&nbsp;&nbsp;regular_expression - The regular expression the value of the referenced</div><div class='line' id='LC1147'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;tabstop is matched on</div><div class='line' id='LC1148'>&nbsp;&nbsp;&nbsp;replacement        - The replacement string, explained in detail below</div><div class='line' id='LC1149'>&nbsp;&nbsp;&nbsp;options            - Options for the regular expression</div><div class='line' id='LC1150'><br/></div><div class='line' id='LC1151'>The options can be any combination of &gt;</div><div class='line' id='LC1152'>&nbsp;&nbsp;&nbsp;g    - global replace</div><div class='line' id='LC1153'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;By default, only the first match of the regular expression is</div><div class='line' id='LC1154'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;replaced. With this option all matches are replaced.</div><div class='line' id='LC1155'>&nbsp;&nbsp;&nbsp;i    - case insensitive</div><div class='line' id='LC1156'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;By default, regular expression matching is case sensitive. With this</div><div class='line' id='LC1157'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;option, matching is done without regard to case.</div><div class='line' id='LC1158'>&nbsp;&nbsp;&nbsp;a    - ascii conversion</div><div class='line' id='LC1159'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;By default, transformation are made on the raw utf-8 string. With</div><div class='line' id='LC1160'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;this option, matching is done on the corresponding ASCII string</div><div class='line' id='LC1161'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;instead, for example &#39;à&#39; will become &#39;a&#39;.</div><div class='line' id='LC1162'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;This option required the python package &#39;unidecode&#39;.</div><div class='line' id='LC1163'><br/></div><div class='line' id='LC1164'>The syntax of regular expressions is beyond the scope of this document. Python</div><div class='line' id='LC1165'>regular expressions are used internally, so the python &#39;re&#39; module can be used</div><div class='line' id='LC1166'>as a guide. See http://docs.python.org/library/re.html.</div><div class='line' id='LC1167'><br/></div><div class='line' id='LC1168'>The syntax for the replacement string is unique. The next paragraph describes</div><div class='line' id='LC1169'>it in detail.</div><div class='line' id='LC1170'><br/></div><div class='line' id='LC1171'><br/></div><div class='line' id='LC1172'>&nbsp;4.7.1 Replacement String:                     *UltiSnips-replacement-string*</div><div class='line' id='LC1173'><br/></div><div class='line' id='LC1174'>The replacement string can contain $no variables, e.g., $1, which reference</div><div class='line' id='LC1175'>matched groups in the regular expression. The $0 variable is special and</div><div class='line' id='LC1176'>yields the whole match. The replacement string can also contain special escape</div><div class='line' id='LC1177'>sequences: &gt;</div><div class='line' id='LC1178'>&nbsp;&nbsp;&nbsp;\u   - Uppercase next letter</div><div class='line' id='LC1179'>&nbsp;&nbsp;&nbsp;\l   - Lowercase next letter</div><div class='line' id='LC1180'>&nbsp;&nbsp;&nbsp;\U   - Uppercase everything till the next \E</div><div class='line' id='LC1181'>&nbsp;&nbsp;&nbsp;\L   - Lowercase everything till the next \E</div><div class='line' id='LC1182'>&nbsp;&nbsp;&nbsp;\E   - End upper or lowercase started with \L or \U</div><div class='line' id='LC1183'>&nbsp;&nbsp;&nbsp;\n   - A newline</div><div class='line' id='LC1184'>&nbsp;&nbsp;&nbsp;\t   - A literal tab</div><div class='line' id='LC1185'><br/></div><div class='line' id='LC1186'>Finally, the replacement string can contain conditional replacements using the</div><div class='line' id='LC1187'>syntax (?no:text:other text). This reads as follows: if the group $no has</div><div class='line' id='LC1188'>matched, insert &quot;text&quot;, otherwise insert &quot;other text&quot;. &quot;other text&quot; is</div><div class='line' id='LC1189'>optional and if not provided defaults to the empty string, &quot;&quot;. This feature</div><div class='line' id='LC1190'>is very powerful. It allows you to add optional text into snippets.</div><div class='line' id='LC1191'><br/></div><div class='line' id='LC1192'><br/></div><div class='line' id='LC1193'>&nbsp;4.7.2 Demos:
